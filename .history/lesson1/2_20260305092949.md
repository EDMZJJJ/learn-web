## 项目亮点

1. 平台化复用和工程化建设

```
-- s 背景：在公司多个后台管理系统开发过程中，发现项目间重复代码多，新项目启动需从头搭建基础模块；代码风格不统一，协作时经常出现格式冲突；部署流程依赖手动操作，发版效率低且容易出错。这些问题导致开发周期长、维护成本高。
-- t 目标：抽离组件，帮助后续同类项目快速开发，协作稳定
-- a 行动：组件的抽离，比如列表、弹窗等业务组件；工程化：eslint+prettier,协助搭建了CI/CD流水线，提升了交付的效率和一致性
-- r 结果：新项目启动时，基础组件实现高效复用，开发周期显著缩短。代码规范落地后，代码审查时间大幅减少，线上缺陷率明显下降。CI/CD 流水线实现部署流程自动化，发布成功率显著提升，团队协作更加顺畅。
```

2. 大屏驾驶舱性能优化

```
-- s 背景：在“浙江省散装水泥专用车辆安全共治系统”中，数字驾驶舱需要同时展示多个ECharts图表，且部分数据需实时刷新。初期页面加载缓慢，切换时间维度时卡顿明显，严重影响监管人员的使用体验。
-- t 目标：优化页面渲染性能，保证大屏场景下的流畅交互。
-- a 行动：ECharts图表按需加载，采用Canvas渲染优化，比如关闭不必要的动画效果（实时刷新的图表关闭动画），对多个图表实例进行复用。将数据聚合计算逻辑放到Web Worker中处理。
-- r 结果：大屏页面首屏加载时间明显缩短，图表切换和刷新操作流畅顺滑，后续被复用至系统中其他类似的数据展示场景，整体用户体验得到提升。
```
