## 项目亮点（2-3个，6分钟）

1. 平台化复用和工程化建设

```
-- s 背景：在公司多个后台管理系统开发过程中，发现项目间重复代码多，新项目启动需从头搭建基础模块；代码风格不统一，协作时经常出现格式冲突；部署流程依赖手动操作，发版效率低且容易出错。这些问题导致开发周期长、维护成本高。
-- t 目标：抽离组件，帮助后续同类项目快速开发，协作稳定
-- a 行动：组件的抽离，比如列表、弹窗等业务组件；工程化：eslint+prettier,协助搭建了CI/CD流水线，提升了交付的效率和一致性
-- r 结果：新项目启动时，基础组件实现高效复用，开发周期显著缩短。代码规范落地后，代码审查时间大幅减少，线上缺陷率明显下降。CI/CD 流水线实现部署流程自动化，发布成功率显著提升，团队协作更加顺畅。
```

2. 大屏驾驶舱性能优化

```
-- s 背景：在“浙江省散装水泥专用车辆安全共治系统”中，数字驾驶舱需要同时展示多个ECharts图表，且部分数据需实时刷新。初期页面加载缓慢，切换时间维度时卡顿明显，严重影响监管人员的使用体验。
-- t 目标：优化页面渲染性能，保证大屏场景下的流畅交互。
-- a 行动：ECharts图表按需加载，减少首屏的初始化成本，采用Canvas渲染优化，针对图表的链路降低渲染开销，还有关闭不必要的动画效果（实时刷新的图表关闭动画），对多个图表实例进行复用，防抖处理窗口resize。将数据聚合计算逻辑放到Web Worker中处理，从主任务当中移出，减少主线程的阻塞
-- r 结果：大屏页面加载时间从2s缩短到1s，打包体积，图表切换和刷新操作流畅顺滑，后续被复用至系统中其他类似的数据展示场景，整体用户体验得到提升。
```

3. 复杂审核流程图的自研渲染

```
-- s 背景：在“浙江省局项目管理系统”中，项目从规划、申报、立项到实施、验收，每个阶段都需要配置不同的审核流程。如果直接使用现成的流程图库（如bpmn.js），难以满足与业务表单深度绑定的需求，且后期维护成本高。
-- t 目标：开发一套可视化的流程配置页面，让业务人员通过可视化操作自主设计审批流程。
-- a 行动：与后端约定标准化的树数据结构，前端配置生成的数据直接存储为树形结构，负责交互映射，灵活表达多级嵌套、条件分支等复杂逻辑。利用Vue递归组件动态解析树数据，根据节点类型渲染对应组件。
-- r 结果：该配置页面上线后，业务人员可独立完成流程设计，流程变更响应时间从数天缩短至小时级，提升了30%的整体效率，自研方案完全贴合业务需求；组件与Vue响应式体系深度集成，代码维护成本低，且团队成员学习成本低。
```
